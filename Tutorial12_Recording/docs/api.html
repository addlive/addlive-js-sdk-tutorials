Recording API

Recording API allows for audio/video capture of AddLive sessions. Using this API one can start, stop recording, list past recordings and get recording status. Once recording is stopped and uploaded, it is possible to get its description. Description contains list of events (users joining and leaving session) and media download URLs.

Every recording has its life-cycle (RECORDING->UPLOADING->READY). When started recording has a state of RECORDING. After stopping recording changes its state to UPLOADING, in this state it is processed, its media files are transcoded and uploaded. Once this process is finished it is marked as READY.

Arguments meaning:

  • signature - signature of a call; in almost all cases signatures is SHA256 digest of concatenation of app_id, scope_id, salt, expires and application API key. The only exception is recording call where signatures is SHA256 digest of app_id, scope_id, user_id, salt, expires and application API key (as in streamer connection details, i.e. with user_id).
  • salt - salt used in signature calculation
  • expires - signature expires after this timestamp
  • app_id - application ID
  • scope_id - scope ID
  • user_id - user ID
  • status - only list recordings with given status, one of “RECORDING”, “ERROR”, “UPLOADING”, “READY”
  • begin - only list recordings started after this timestamp
  • end - only list recordings started before this timestamp
  • limit - show at most this number of recordings
  • offset - offset of recordings list
  • order_by - order recordings list by this field, one of: 'id', 'app_id', 'scope_id', 'starts_at', 'status', 'last_activity'
  • descending - reverse recordings list sorting order
  • urls_ttl - generated media download URLs should not live longer than this number of seconds (default: 3600)

Arguments are passed in query string (GET) or as form-urlencoded data (POST).

Results:

Every call results in a reply encoded in JSON. Results are packed in a structure:

{"message": null, "result": ..., "success": true, "error": false}

success and error flags represent call status, message contains more details in case of error. If method returns result it’s passed in result field.

Example:

$ curl -d app_id=1 -d scope_id=some_scope -d salt=BzBgl -d signature=32a55c80e6aa8cfea97ea65e7d0736e439ef7051e1f6c10e32793ba0887f9250 -d user_id=1 -d expires=1412630689 'http://recording-beta.addlive.io/recordings/'

{"message": null, "result": {"status": "RECORDING", "last_activity": 1412627543, "scope_id": "some_scope", "started_at": 1412627543, "app_id": 1, "files": [], "id": "0c7ceaf4-bac9-4da1-9d1b-30bb8ca63d6b"}, "success": true, "error": false}

$ curl -X POST 'http://recording-beta.addlive.io/recordings/0c7ceaf4-bac9-4da1-9d1b-30bb8ca63d6b/stop'

{"message": null, "result": {"status": "UPLOADING", "last_activity": 1412627565, "scope_id": "some_scope", "started_at": 1412627543, "app_id": 1, "files": [], "id": "0c7ceaf4-bac9-4da1-9d1b-30bb8ca63d6b"}, "success": true, "error": false}
